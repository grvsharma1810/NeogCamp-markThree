So, JSX stands for Javascript XML. It is a beautiful fusion of HTML and Javascript. It enables us to write Javascript in HTML or vice-versa. JSX in mainly used in React to render the components for Virtual DOM (React has a concept of Virtual DOM). So, what it does is, it generates a lightweight view of the components of DOM (which is then mapped to actual DOM components). Here, is an example - 

    class comp extends Component{
        getName(){
            return "Gaurav";
        }
        render(){
            return(
                <div%gt
                    <h1>{this.getName()}</h1>
                </div%gt
                );
        }
    }
Can you see what the render function is returning? That's JSX. Inside <h1%gt tag, getName() method is called. Wherever we need to write javascript, we write it inside curly braces. So, the render function will paint the name "Gaurav" on the screen. JSX uses ES6 features and internally it is converted to pure javascript code by something called as BABEL. Actually BABEL is a compiler. Each tag calls a function React.createElement() internally. This code will be converted into :
React.createElement("div", null, React.createElement("h1", null, (void 0).getName())); as you can see here: https://babeljs.io/repl Writing JSX : -
It is really easy to write JSX. Here are some rules you have to follow :
  1. The class keyword used in HTML tags like <div class="something"> is replaced by className.
  2. for, while, if-else can't be used, only expressions are allowed in javascript.
  3. Dataflow is taken care of using Props (an abbreviation of Properties).
